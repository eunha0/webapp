## 【발명의 명칭】

AI 기반 논술 자동 채점 및 학년별 맞춤형 피드백 생성 시스템

## 【기술분야】

본 발명은 인공지능 기반 교육 평가 시스템에 관한 것으로, 더욱 상세하게는 대규모 언어 모델(LLM)을 활용하여 학생 논술을 자동으로 채점하고, 학년 수준에 따라 톤이 조정된 상세한 맞춤형 피드백을 생성하며, 다양한 입력 형식(텍스트, 이미지, PDF)을 지원하고, 학생의 성장을 추적하는 통합 논술 평가 시스템에 관한 것이다.

## 【배경기술】

### 종래 기술의 문제점

#### 1. 수동 채점의 한계
교사가 학생 논술을 채점하는 데 많은 시간이 소요된다. 예를 들어, 30명 학급의 논술을 채점하는 데 평균 5-8시간이 소요되며, 이는 교사의 업무 부담을 가중시키고 다른 교육 활동에 투입할 시간을 감소시킨다.

#### 2. 평가 일관성 부족
교사의 주관성, 피로도, 채점 순서 등에 따라 동일한 논술에 대한 평가가 달라질 수 있어, 학생들에게 공정하지 못한 평가 결과를 제공할 위험이 있다.

#### 3. 피드백의 제한성
시간 부족으로 인해 교사가 제공하는 피드백이 피상적이거나 일반적인 수준에 그치는 경우가 많으며, 학생 개개인의 학년 수준이나 강약점을 고려한 맞춤형 피드백 제공이 어렵다.

#### 4. 성장 추적의 어려움
학생이 제출한 여러 논술에 걸쳐 개선 사항을 체계적으로 추적하고 기록하는 것이 어려우며, 이로 인해 학생의 학습 진전도를 파악하기 어렵다.

#### 5. 입력 형식의 제한
기존 자동 채점 시스템은 주로 텍스트 입력만 지원하며, 손글씨 답안이나 스캔된 문서를 자동으로 인식하여 채점하는 기능이 부족하다.

### 기존 기술의 한계

기존의 자동 논술 채점(AES: Automated Essay Scoring) 시스템들은 다음과 같은 한계를 갖고 있다:

1. **EssayGrader.ai**: 영어 중심, 한글 지원 부족, 학년별 톤 조정 부재
2. **Grammarly**: 문법 교정 중심, 논술 구조 및 논리성 평가 부족
3. **Turnitin**: 표절 검사 중심, 상세 피드백 부족
4. **기타 시스템**: 단순 점수 산출, 개선 방법 제시 부족, 재제출 추적 부재

### 본 발명의 필요성

따라서, 교사의 채점 시간을 획기적으로 단축하면서도 일관성 있고 상세한 평가를 제공하고, 학년 수준에 맞는 맞춤형 피드백을 생성하며, 다양한 입력 형식을 지원하고, 학생의 성장을 체계적으로 추적할 수 있는 통합 논술 평가 시스템의 개발이 절실히 요구된다.

## 【발명의 내용】

### 【해결하고자 하는 과제】

본 발명의 목적은 상기와 같은 종래 기술의 문제점을 해결하기 위한 것으로, 구체적으로는:

1. **자동 채점**: AI를 활용하여 논술을 자동으로 채점하고 루브릭 기준별로 점수를 부여
2. **맞춤형 피드백**: 학년 수준(초등/중등/고등)에 따라 톤이 조정된 상세 피드백 생성
3. **구조화된 피드백**: 긍정적 피드백, 개선 영역, 구체적 제안의 3단계 구조화
4. **다중 입력 지원**: 텍스트, 이미지(OCR), PDF(텍스트/이미지) 등 다양한 형식 지원
5. **성장 추적**: 재제출 시 개선율 자동 계산 및 학생 진전도 추적
6. **교사 검토 지원**: Split-Screen 인터페이스를 통한 효율적 검토 및 수정
7. **일괄 처리**: 다수의 논술을 동시에 채점하고 결과를 통합 출력

#### (1) 시스템 구성

##### A. 학년별 맞춤형 피드백 생성 알고리즘

```
입력: 논술 텍스트, 학년 수준, 루브릭 기준, 채점 점수
출력: 학년 맞춤 피드백

1. 학년 수준 분석:
   IF 학년 = "초등학교":
      톤 = "친근하고 격려적"
      어휘 = "쉬운 단어 사용"
      예시 = "구체적이고 시각적"
   ELSE IF 학년 = "중학교":
      톤 = "중간 수준의 격식"
      어휘 = "표준 어휘 사용"
      예시 = "실생활 관련"
   ELSE IF 학년 = "고등학교":
      톤 = "학술적이고 심화된"
      어휘 = "전문 용어 포함"
      예시 = "추상적이고 분석적"

2. 기준별 피드백 생성:
   FOR EACH 루브릭_기준 IN 루브릭_목록:
      점수 = 채점_결과[루브릭_기준]
      
      IF 점수 >= 3:
         긍정적_피드백 = 생성_강점_피드백(논술, 기준, 톤, 어휘)
      
      IF 점수 <= 2:
         개선_영역 = 생성_약점_분석(논술, 기준, 톤, 어휘)
      
      구체적_제안 = 생성_단계별_제안(논술, 기준, 점수, 톤, 어휘, 예시)
      
      기준별_피드백.추가({
         기준명: 루브릭_기준,
         점수: 점수,
         긍정적_피드백: 긍정적_피드백,
         개선_영역: 개선_영역,
         구체적_제안: 구체적_제안
      })

3. 전체 요약 생성:
   총점 = SUM(각_기준_점수)
   강점_목록 = 추출_강점(기준별_피드백, 상위_N개)
   약점_목록 = 추출_약점(기준별_피드백, 하위_N개)
   우선_개선사항 = 결정_우선순위(약점_목록, 학년_수준)
   종합_의견 = 생성_종합_평가(논술, 총점, 강점, 약점, 톤, 어휘)
   다음_단계 = 생성_학습_조언(약점_목록, 우선_개선사항, 학년_수준)

4. 피드백 구조화:
   RETURN {
      기준별_피드백: 기준별_피드백,
      전체_요약: {
         총점: 총점,
         강점: 강점_목록,
         약점: 약점_목록,
         종합_의견: 종합_의견,
         우선_개선사항: 우선_개선사항,
         다음_단계: 다음_단계
      }
   }
```

##### B. 다중 형식 입력 처리 알고리즘

```
입력: 파일 (이미지 또는 PDF)
출력: 추출된 논술 텍스트

1. 파일 타입 감지:
   mime_type = 파일.mime_type
   
2. 이미지 처리:
   IF mime_type IN ["image/jpeg", "image/png", "image/webp"]:
      텍스트 = OCR_처리(파일, API="Google_Vision")
      RETURN 텍스트

3. PDF 처리:
   IF mime_type = "application/pdf":
      A. 텍스트 기반 PDF 확인:
         시도 {
            텍스트 = PDF_텍스트_추출(파일, 라이브러리="PDF.js")
            IF LEN(텍스트) > 최소_문자_수:
               RETURN 텍스트
         } 예외처리 {
            // 텍스트 추출 실패 시 OCR로 전환
         }
      
      B. 이미지 기반 PDF 처리:
         이미지_목록 = PDF를_이미지로_변환(파일)
         전체_텍스트 = ""
         FOR EACH 이미지 IN 이미지_목록:
            페이지_텍스트 = OCR_처리(이미지, API="Google_Vision")
            전체_텍스트 += 페이지_텍스트 + "\n"
         RETURN 전체_텍스트

4. 에러 처리:
   ELSE:
      THROW "지원하지 않는 파일 형식"
```

##### C. 성장 추적 및 개선율 계산 알고리즘

```
입력: 학생_ID, 과제_ID, 현재_점수
출력: 개선율, 성장_기록

1. 기존 제출 이력 조회:
   제출_이력 = DB.조회(학생_ID, 과제_ID, 정렬="제출일_역순")
   
2. 개선율 계산:
   IF LEN(제출_이력) > 1:
      이전_점수 = 제출_이력[1].점수
      현재_점수 = 제출_이력[0].점수
      
      IF 이전_점수 > 0:
         개선율 = ((현재_점수 - 이전_점수) / 이전_점수) × 100
      ELSE:
         개선율 = 0
   ELSE:
      개선율 = NULL  // 첫 제출

3. 성장 기록 업데이트:
   성장_기록 = {
      제출_횟수: LEN(제출_이력),
      최고_점수: MAX(제출_이력.점수),
      최근_점수: 현재_점수,
      개선율: 개선율,
      마지막_제출일: 현재_시간
   }
   DB.업데이트(학생_ID, 과제_ID, 성장_기록)

4. 추세 분석 (선택):
   IF LEN(제출_이력) >= 3:
      추세 = 회귀_분석(제출_이력.점수)
      IF 추세 > 임계값:
         상태 = "지속적 개선"
      ELSE IF 추세 < -임계값:
         상태 = "주의 필요"
      ELSE:
         상태 = "안정적"
      성장_기록.추세_상태 = 상태

   RETURN 개선율, 성장_기록
```

##### D. Split-Screen 검토 인터페이스 알고리즘

```
입력: 제출물_ID
출력: 검토_인터페이스 데이터

1. 데이터 로드:
   제출물 = DB.조회_제출물(제출물_ID)
   피드백 = DB.조회_피드백(제출물_ID)
   과제_정보 = DB.조회_과제(제출물.과제_ID)

2. 화면 분할 구성:
   좌측_패널 = {
      타입: "읽기_전용",
      내용: 제출물.논술_텍스트,
      스크롤: "독립",
      헤더: {
         학생명: 제출물.학생_이름,
         제출일: 제출물.제출일,
         과제명: 과제_정보.제목
      }
   }
   
   우측_패널 = {
      타입: "편집_가능",
      내용: 피드백,
      스크롤: "독립",
      필드: [
         {이름: "총점", 타입: "숫자", 범위: [0, 10]},
         {이름: "종합_평가", 타입: "텍스트영역"},
         FOR EACH 기준 IN 피드백.기준별_피드백:
            {이름: 기준.기준명 + "_점수", 타입: "숫자", 범위: [1, 4]},
            {이름: 기준.기준명 + "_긍정", 타입: "텍스트영역"},
            {이름: 기준.기준명 + "_개선", 타입: "텍스트영역"},
            {이름: 기준.기준명 + "_제안", 타입: "텍스트영역"}
      ]
   }

3. 실시간 동기화:
   ON 우측_패널.값_변경:
      임시_저장(제출물_ID, 변경된_필드, 새_값)
   
   ON 저장_버튼_클릭:
      수정된_피드백 = 수집_모든_필드_값(우측_패널)
      DB.업데이트_피드백(제출물_ID, 수정된_피드백)
      알림("피드백이 저장되었습니다")

4. 출력 기능:
   ON 출력_버튼_클릭:
      출력_문서 = 생성_출력_문서({
         과제_정보: 과제_정보,
         제출물: 제출물,
         피드백: 수정된_피드백
      })
      새_창_열기(출력_문서, 인쇄_최적화=TRUE)

   RETURN {좌측_패널, 우측_패널}
```

#### (3) 시스템 워크플로우

```
[교사] → 과제 생성 (제목, 설명, 루브릭, 제시문)
       → 액세스 코드 생성
       → 학생에게 공유
       
[학생] → 액세스 코드 입력
       → 과제 확인 (제목, 설명, 루브릭)
       → 답안 작성 및 제출
       
[시스템] → 답안 수신
         → 파일 형식 감지 및 텍스트 추출 (필요시 OCR)
         → AI 자동 채점 (루브릭 기준별)
         → 학년별 맞춤형 피드백 생성
         → DB 저장
         
[교사] → 채점 결과 확인
       → Split-Screen 검토
       → 피드백 수정 (선택)
       → 저장 또는 출력
       
[학생] → 피드백 확인
       → 개선 사항 학습
       → 재제출 (선택)
       
[시스템] → 개선율 자동 계산
         → 성장 기록 업데이트
```

### 【발명의 효과】

본 발명에 따른 AI 기반 논술 자동 채점 및 학년별 맞춤형 피드백 생성 시스템은 다음과 같은 효과가 있다:

#### 1. 시간 효율성
- 전통적 채점 방식 대비 **90% 시간 단축** (30명 학급 기준: 5시간 → 30분)
- 교사가 교육 콘텐츠 개발 및 학생 상담 등 핵심 활동에 집중 가능

#### 2. 평가 일관성
- AI 기반 자동 채점으로 **평가 기준의 일관성** 확보
- 인간 채점자 수준의 신뢰도(QWK 0.61+) 달성 목표
- 채점자의 주관성, 피로도에 영향받지 않는 공정한 평가

#### 3. 맞춤형 교육
- **학년별로 조정된 톤**(초등: 친근함, 중등: 중간, 고등: 학술적)으로 학생 이해도 향상
- **3단계 구조화된 피드백**(긍정 + 개선 + 제안)으로 명확한 개선 방향 제시
- 학생별 강약점을 고려한 개별 맞춤 피드백

#### 4. 입력 유연성
- **다중 형식 지원**(텍스트, 이미지, PDF)으로 손글씨 답안도 처리 가능
- **OCR 기술 통합**으로 이미지 및 스캔 문서 자동 인식
- 교사 및 학생의 사용 편의성 극대화

#### 5. 성장 추적
- **재제출 시 개선율 자동 계산**으로 학생의 학습 진전도 가시화
- 장기적인 성장 추적 데이터로 교육 전략 수립 지원
- 학생 동기 부여 및 자기 주도 학습 촉진

#### 6. 교사 역량 강화
- **Split-Screen 검토 인터페이스**로 효율적인 피드백 수정
- AI가 초안을 제공하고 교사가 최종 검토함으로써 **AI와 인간의 협업** 실현
- 교육 품질 향상 및 교사 만족도 증가

#### 7. 확장 가능성
- 클라우드 기반 아키텍처(Cloudflare Workers)로 **글로벌 확장** 용이
- 다양한 교과목(국어, 사회, 과학 등) 및 평가 유형에 적용 가능
- LMS 통합, 모바일 앱 등 다양한 플랫폼으로 확장 가능

## 【발명을 실시하기 위한 구체적인 내용】

### 실시예 1: 고등학생 논술 채점 및 피드백 생성

#### 입력
- **논술 텍스트**: "인공지능의 발달이 인간의 고용에 미치는 영향에 대해 논하시오." (1000자)
- **학년 수준**: 고등학교 2학년
- **루브릭 기준**:
  1. 논리성 (주장의 명확성 및 논리적 흐름)
  2. 근거의 타당성 (충분하고 설득력 있는 근거)
  3. 반론 및 재반박 (반대 의견 고려 및 재반박)
  4. 글의 구조 (서론-본론-결론 구성)

#### 처리 과정

**1단계: 학년 수준 분석**
```python
학년 = "고등학교"
톤 = "학술적이고 심화된"
어휘 = "전문 용어 포함"
예시 = "추상적이고 분석적"
```

**2단계: AI 자동 채점**
- 논리성: 4/4점 (우수)
- 근거의 타당성: 3/4점 (양호)
- 반론 및 재반박: 2/4점 (발전 필요)
- 글의 구조: 3/4점 (양호)
- **총점**: 12/16점 → 7.5/10점

**3단계: 기준별 피드백 생성**

**기준 1: 논리성 (4/4점)**
- **긍정적 피드백**: "논증의 구조가 탄탄하며, 주장과 근거 간의 연결이 논리적으로 명확합니다. 특히 'AI 기술 발달 → 일자리 변화 → 사회적 대응 필요'라는 논리적 흐름이 자연스럽게 전개되었습니다."
- **개선 영역**: (없음, 만점)
- **구체적 제안**: "현재 수준을 유지하면서, 더 복잡한 논증 구조(예: 다층적 논증, 조건부 논증)를 시도해 보세요."

**기준 2: 근거의 타당성 (3/4점)**
- **긍정적 피드백**: "통계 자료와 사례를 적절히 활용하여 주장을 뒷받침했습니다. 특히 '옥스퍼드 대학 연구' 인용은 설득력을 높였습니다."
- **개선 영역**: "일부 근거가 일반화되어 있어 구체성이 부족합니다. 예를 들어, '많은 일자리가 사라질 것'보다는 구체적 통계나 업종별 분석이 필요합니다."
- **구체적 제안**:
  1. "세부 통계 추가: '제조업 30%, 사무직 15% 감소 예상' 등 구체적 수치 제시"
  2. "전문가 의견 인용: 경제학자나 산업 전문가의 전망 추가"
  3. "국내외 사례 비교: 한국과 다른 국가의 사례를 비교 분석"

**기준 3: 반론 및 재반박 (2/4점)**
- **긍정적 피드백**: "반론을 제시하려는 시도가 보입니다."
- **개선 영역**: "반론에 대한 재반박이 미흡합니다. '일부는 AI가 새로운 일자리를 창출한다고 주장한다'고 언급했지만, 이에 대한 충분한 반박이 없습니다."
- **구체적 제안**:
  1. "강한 반론 설정: 반대 입장의 가장 강력한 주장을 먼저 제시하세요."
  2. "증거 기반 재반박: '하지만 새 일자리는 고숙련 인력에 한정되며, 저숙련 노동자는 배제된다'와 같이 구체적 근거로 반박하세요."
  3. "양보와 재강조: 일부 타당성을 인정하면서도 본인의 주장이 더 설득력 있음을 강조하세요."

**기준 4: 글의 구조 (3/4점)**
- **긍정적 피드백**: "서론-본론-결론의 3단 구성이 명확하며, 각 단락의 주제가 분명합니다."
- **개선 영역**: "결론 부분이 다소 짧고 반복적입니다. 서론에서 제시한 문제에 대한 해결책이나 통찰을 추가하면 더 완성도 높은 논술이 될 것입니다."
- **구체적 제안**:
  1. "결론 확장: 본론에서 논의한 내용을 바탕으로 구체적 정책 제안이나 사회적 방향성 제시"
  2. "미래 전망: 단순 요약을 넘어 'AI 시대의 인간의 역할'에 대한 통찰 추가"
  3. "독자 행동 유도: '우리는 어떻게 대비해야 하는가?'와 같은 질문으로 독자의 참여 유도"

**4단계: 전체 요약**
- **총점**: 7.5/10점 (75%)
- **강점**:
  1. 논리적 구조가 탄탄하고 흐름이 자연스럽습니다
  2. 통계 자료와 사례를 적절히 활용했습니다

## 【요약서】

### 【요약】

본 발명은 AI 기반 논술 자동 채점 및 학년별 맞춤형 피드백 생성 시스템에 관한 것이다. 본 시스템은 대규모 언어 모델(LLM)을 활용하여 학생의 논술을 자동으로 채점하고, 학년 수준(초등/중등/고등)에 따라 톤이 조정된 상세한 맞춤형 피드백을 생성한다. 

핵심 특징은 다음과 같다:
1. **학년별 톤 조정**: 초등학생에게는 친근하고 격려적인 톤, 고등학생에게는 학술적이고 심화된 톤으로 피드백 제공
2. **3단계 구조화 피드백**: 긍정적 피드백 + 개선 영역 + 구체적 제안의 체계적 구성
3. **다중 입력 지원**: 텍스트, 이미지(OCR), PDF 등 다양한 형식 지원
4. **성장 추적**: 재제출 시 개선율 자동 계산 및 학생 진전도 추적
5. **Split-Screen 검토**: 교사가 학생 답안과 피드백을 동시에 확인하며 수정 가능

본 발명은 교사의 채점 시간을 90% 단축하고, 학생에게 즉각적이고 맞춤형 피드백을 제공하여 교육의 질을 혁신적으로 향상시킨다.
